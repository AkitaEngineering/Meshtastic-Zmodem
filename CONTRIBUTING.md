# Contributing

Thanks for contributing to Akita Meshtastic ZModem.

- Code style: follow existing project style (spaces for indentation, descriptive names).
- Tests: provide small test sketches or PlatformIO environments where possible.
- Pull requests: open feature branches, include concise description and testing steps.
- Issues: include device type, firmware revision, and serial logs when reporting bugs.

Before opening a PR:

1. Run a local build for your target board.
2. Verify no new warnings or regressions.
3. Add a short entry to `CHANGELOG.md` describing the change.
